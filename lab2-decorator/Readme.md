# Лабораторная работа: паттерн «Декоратор» для курсов валют ЦБ РФ

## Цель работы
Реализовать получение курсов валют Центробанка РФ в формате JSON и расширить базовую функциональность с помощью шаблона проектирования **Decorator**: добавить возможность преобразования данных в **CSV** и **YAML**, а также сохранения результата в файл соответствующего типа. Дополнительно — обеспечить качество кода по принципам DAST/DATS: тесты, аннотации типов, документация и стиль.

## Используемые технологии и библиотеки
- **requests** — HTTP-запрос к API ЦБ РФ.
- **json** — сохранение результата базового компонента в `.json`.
- **csv** (встроенная библиотека) — формирование и запись `.csv`.
- **PyYAML (yaml)** — формирование и запись `.yaml`.
- **pytest + requests-mock** — модульное тестирование без реальных сетевых запросов.
- **typing** — аннотации типов по PEP-484.
- **abc** — интерфейс/абстракция через `ABC` и `@abstractmethod`.

## Краткая постановка задачи
1. Определить интерфейс (абстрактный базовый класс), который задаёт:
   - метод получения данных о валютах;
   - метод сохранения результата в файл.
2. Реализовать базовый компонент:
   - выполняет запрос к API ЦБ РФ;
   - возвращает словарь валют из ответа (`Valute`);
   - умеет сохранять данные в JSON-файл.
3. Реализовать декоратор и конкретные декораторы:
   - общий декоратор хранит ссылку на объект интерфейса и делегирует вызовы;
   - `ConvertToCSV` преобразует данные в табличный вид (список словарей) и сохраняет как CSV;
   - `ConvertToYAML` возвращает данные в виде словаря и сохраняет как YAML.
4. Написать тесты:
   - 2 теста для базового компонента;
   - 2 теста для CSV-декоратора;
   - 2 теста для YAML-декоратора.

## Архитектура решения
Решение построено вокруг паттерна **Decorator**:

- **Currencies (ABC)** — контракт (интерфейс), единая точка согласования поведения:
  - `getCurrencies()` — получение данных;
  - `createFile()` — сохранение данных в файл.

- **GetCurrencies** — базовый компонент:
  - получает JSON с API ЦБ РФ;
  - валидирует, что в ответе есть ключ `Valute`;
  - возвращает словарь валют (`dict`);
  - сохраняет результат в JSON.

- **Convert** — базовый декоратор:
  - принимает объект `Currencies`;
  - по умолчанию делегирует методы `getCurrencies()` и `createFile()` базовому компоненту;
  - служит основой для конкретных декораторов.

- **ConvertToCSV** — конкретный декоратор:
  - преобразует структуру `dict валюта -> поля` в `list[dict]`, добавляя колонку `ID_Валюты`;
  - формирует `_columns` и `_rows` для последующей записи;
  - сохраняет данные в CSV с разделителем `;`.

- **ConvertToYAML** — конкретный декоратор:
  - получает словарь валют и сохраняет его как YAML;
  - использует `allow_unicode=True` для корректной записи кириллицы.

Таким образом, расширение логики достигается **без изменения** базового класса получения валют, что соответствует идее паттерна «Декоратор»: добавлять поведение динамически через композицию.

## Обработка ошибок и валидация
В реализации предусмотрены типовые проверки:
- Ошибка структуры ответа API: если отсутствует `Valute`, выбрасывается `KeyError`.
- Ошибка порядка вызовов: сохранение файла возможно только после получения данных:
  - для базового компонента проверяется наличие `self.data`;
  - для CSV — наличие `_rows` и `_columns`;
  - для YAML — наличие `self.data`.

## Тестирование
Тестирование выполнено модульно с изоляцией от сети:
- Используется `requests_mock`, чтобы подменять ответ API ЦБ РФ.
- Набор проверок включает:
  - базовый успешный сценарий получения данных;
  - базовый сценарий с некорректным ответом (нет `Valute`);
  - корректность преобразования JSON → CSV (структура списка строк);
  - создание CSV-файла и проверка содержимого (наличие заголовков и разделителя);
  - корректность получения данных через YAML-декоратор;
  - создание YAML-файла и проверка ключевых фрагментов.

## Соответствие требованиям качества (DAST / DATS)
- **Тесты:** реализованы тестовые сценарии для базового функционала и каждого декоратора (по 2 на компонент).
- **Аннотации типов (PEP-484):** применены в сигнатурах методов и структурах возвращаемых данных.
- **Документация (PEP-257):** классы и методы снабжены docstring-комментариями.
- **Стиль (PEP-8):** структура кода соответствует общим рекомендациям по оформлению (именование, отступы, читаемость).

## Вывод
В ходе лабораторной работы реализована расширяемая система получения курсов валют через API ЦБ РФ с применением паттерна **Decorator**. Базовый компонент отвечает за извлечение и валидацию данных, а декораторы добавляют преобразование результата в форматы **CSV** и **YAML** и запись в файлы. Решение сопровождается модульными тестами и соответствует требованиям по аннотациям типов, документации и стилю.
